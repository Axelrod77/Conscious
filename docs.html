Conscious Documentation

Conscious is the voice-first input layer for enterprise SaaS. It makes every <input> and <textarea> in your React app voice-aware — without replacing your UI or building a chatbot.

The user clicks a field, presses Fn or Ctrl, and speaks. The transcription commits to the field with a Confidence Score. That's the whole product.

Installation

npm install @conscious/react

Requires React 17+. Works with Next.js, Vite, Create React App, and any standard React setup.

Quick Start

Three lines makes your entire app voice-conscious. Wrap your root component with ConsciousProvider and every input and textarea becomes voice-aware automatically.

import { ConsciousProvider } from '@conscious/react'

// app.tsx (or _app.tsx for Next.js)
<ConsciousProvider proxyUrl="/api/voice">
  <App />
</ConsciousProvider>

That's it. Every <input type="text">, <input type="search">, and <textarea> now responds to the trigger key.

Proxy Setup

Conscious requires a backend proxy so your STT API key never touches the client. This is a security requirement, not optional. Here's the minimal Next.js API route:

// pages/api/voice.ts (Next.js)
import { createConsciousProxy } from '@conscious/proxy'

export default createConsciousProxy({
  apiKey: process.env.WISPR_FLOW_API_KEY,    // Never on the client
  allowedOrigins: ['https://yourapp.com'],
})

For Express, Fastify, or other Node servers — see the proxy adapters guide. The hosted API option is available on the Growth plan with no proxy setup required.

ConsciousProvider

The root provider that initialises the voice engine and manages WebSocket connections.

<ConsciousProvider
  proxyUrl="/api/voice"               // Required: your proxy endpoint
  triggerKey="fn"                    // "fn" | "ctrl" | "alt" — default: "fn"
  ccsThreshold={90}                   // Auto-commit threshold — default: 90
  language="en-US"                   // BCP 47 language tag — default: "en-US"
  onError={handleError}               // Error callback
  connectionTimeoutMs={2000}          // Proxy timeout — default: 2000
>
  <App />
</ConsciousProvider>

ConsciousInput

For cases where you want voice input on specific fields only, or need to add context hints:

import { ConsciousInput } from '@conscious/react'

<ConsciousInput
  fieldLabel="Account Notes"          // Improves accuracy by ~12%
  fieldType="notes"                  // "text" | "notes" | "email" | "number" | "date" | "search"
  placeholder="Type or press Fn..."
  onChange={handleChange}
  value={value}
/>

useConscious hook

For building custom voice UI or accessing the session state:

import { useConscious } from '@conscious/react'

function MyField() {
  const { isListening, ccs, transcript, startListening, stopListening } = useConscious()

  return (
    <div>
      {isListening && <span>Listening... (CCS: {ccs}%)</span>}
    </div>
  )
}

Conscious Confidence Score (CCS)

Every transcription returns a CCS between 0–100. The score drives the commit behaviour:

  • ≥90 (default) — Auto-commits to the field. Field border flashes green.
  • 70–89 — Shows inline suggestion: "Did you mean X?" with Accept / Edit.
  • <70 — Shows disambiguation panel with 3 ranked options.

You control the auto-commit threshold via ccsThreshold on the provider. Medical or compliance apps typically set this to 95+.

Field Type Detection

Conscious auto-detects field types from the type attribute and fieldLabel prop, and adjusts transcription accordingly:

  • number / tel — Outputs digits: "five" → "5"
  • email — Formats addresses: "john at acme dot com" → "john@acme.com"
  • date — Parses natural language: "next Tuesday" → "2025-03-04"
  • search — Strips punctuation, outputs clean query strings
  • text / notes — Preserves natural sentence structure and punctuation

Error Codes

Handle errors via the onError prop on ConsciousProvider:

const handleError = (error) => {
  switch (error.code) {
    case 'MIC_DENIED':        // User blocked microphone access
    case 'STT_ERROR':         // Backend proxy or Wispr Flow failure
    case 'AUTH_FAILED':       // Proxy authentication rejected
    case 'CONNECTION_TIMEOUT': // Proxy failed to respond in time
    case 'QUOTA_EXCEEDED':    // Monthly session limit reached
  }
}

Trigger Keys

The default trigger is Fn (Function key), inspired by Wispr Flow's interaction model. You can change it:

<ConsciousProvider triggerKey="ctrl"> // Left Ctrl
<ConsciousProvider triggerKey="alt">  // Alt / Option
<ConsciousProvider triggerKey="fn">   // Fn key (default)

On macOS, Fn requires the user to set "Press Fn key to: Do Nothing" in System Preferences → Keyboard — otherwise Fn toggles function keys. Ctrl or Alt avoids this friction for cross-platform apps.

CRM Integration Guide

For CRM apps (Salesforce-style), the highest-impact fields to voice-enable are:

  • Call notes and activity logs
  • Deal descriptions and next steps
  • Account notes and contact context
  • Support ticket descriptions

Set fieldLabel accurately — it's used by the STT engine to improve accuracy. "Account Notes" outperforms a generic unlabelled field by ~12% CCS on average.

Security Architecture

Audio travels from the browser to your backend proxy via an encrypted WebSocket. Your proxy relays it to the STT provider. The API key lives only on your server. End users and their network inspectors never see a third-party domain.

Conscious stores zero bytes of audio after processing. The transcript is returned to the client and immediately discarded from our infrastructure. Nothing persists.

FAQ

Does this work with controlled components?
Yes. Conscious fires the native change and input events, so React controlled components (value + onChange) work without any modification.

What happens if the microphone is denied?
Conscious catches the MIC_DENIED error and calls your onError handler. The field falls back to normal typing with no visual change for the user.

Can I use Conscious in a non-React app?
Not yet. React is the only supported framework in Phase 1. A vanilla JS adapter and Vue wrapper are on the Phase 2 roadmap.

Is the 95% accuracy guarantee per-session or per-word?
It's measured as Word Error Rate over a rolling 30-day window across all sessions in your workspace. If the aggregate WER exceeds 5%, that month is refunded.
