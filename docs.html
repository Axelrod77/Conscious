Conscious Documentation

Conscious is the voice-first input layer for enterprise SaaS. It eliminates the Input Tax — the 2–3 hours per day your users spend typing into forms — by making every <input> and <textarea> voice-aware.

The interaction is simple: click a field, press Fn or Ctrl, speak, stop. The transcription commits with a Confidence Score. That's the entire product surface area.

Installation

npm install @conscious/react

Requires React 17+. Works with Next.js, Vite, Create React App, Remix, and any standard React environment.

Quick Start

Wrap your app root with ConsciousProvider. Every <input> and <textarea> in the tree becomes voice-aware automatically — no per-component changes.

import { ConsciousProvider } from '@conscious/react'

// In your app root (App.tsx, _app.tsx, main.tsx, etc.)
export default function App() {
  return (
    <ConsciousProvider proxyUrl="/api/voice">
      <YourApp />
    </ConsciousProvider>
  )
}

Proxy Setup

Conscious requires a backend proxy so your STT API key never reaches the browser. This is a security requirement — not optional. Here's the minimal implementation for Next.js:

// pages/api/voice.ts  (or app/api/voice/route.ts for App Router)
import { createConsciousProxy } from '@conscious/proxy'

export default createConsciousProxy({
  apiKey: process.env.WISPR_FLOW_API_KEY,   // Never exposed to client
  allowedOrigins: ['https://yourapp.com'],
})

For Express, Fastify, or other runtimes, see the proxy adapter docs. If you're on the Growth or Enterprise plan, you can use the hosted API option and skip this setup entirely.

ConsciousProvider

The root provider that initialises the voice engine, manages WebSocket connections, and exposes context to all child inputs.

<ConsciousProvider
  proxyUrl="/api/voice"        // Required — your backend proxy endpoint
  triggerKey="fn"              // "fn" | "ctrl" | "alt" | "meta"  (default: "fn")
  ccsThreshold={90}            // Auto-commit threshold 0–100 (default: 90)
  language="en-US"             // BCP 47 language tag (default: "en-US")
  onError={handleError}        // (error: ConsciousError) => void
  connectionTimeoutMs={2000}   // Proxy connection timeout ms (default: 2000)
  debug={false}                // Log internal events to console
>
  <App />
</ConsciousProvider>

ConsciousInput

For granular control — voice input on specific fields only, or custom context hints that improve accuracy:

import { ConsciousInput } from '@conscious/react'

<ConsciousInput
  fieldLabel="Account Notes"       // Passed to STT engine — improves accuracy ~12%
  fieldType="notes"                // "text"|"notes"|"email"|"number"|"date"|"search"
  placeholder="Type or press Fn..."
  value={value}
  onChange={e => setValue(e.target.value)}
/>

useConscious hook

For building custom voice UI, monitoring session state, or triggering voice programmatically:

import { useConscious } from '@conscious/react'

function CustomVoiceButton() {
  const {
    isListening,     // boolean
    ccs,             // number | null — current Confidence Score
    transcript,      // string — current draft transcription
    startListening,  // () => void
    stopListening,   // () => void
    isConnected,     // boolean — WebSocket proxy connected
  } = useConscious()

  return (
    <button onClick={isListening ? stopListening : startListening}>
      {isListening ? `Listening… (CCS: ${ccs}%)` : 'Start voice input'}
    </button>
  )
}

Conscious Confidence Score (CCS)

Every transcription returns a CCS from 0–100. The score drives commit behaviour:

  • ≥90 (configurable) — Auto-commits instantly. Field border flashes green. Zero user friction.
  • 70–89 — Inline suggestion: "Did you mean Acme Corp?" — one tap to accept or edit.
  • <70 — Disambiguation panel: three ranked options. The user's choice trains the vocabulary model.

Set ccsThreshold on the provider to tune for your domain. Medical and compliance apps typically set 95+. Internal tools can go as low as 75.

Field Type Detection

Conscious auto-detects field types from the HTML type attribute and the fieldLabel prop, and normalises output accordingly:

  • number / tel — Converts spoken numbers: "five hundred" → "500"
  • email — Parses addresses: "john at acme dot com" → "john@acme.com"
  • date — Resolves natural language: "next Tuesday" → "2025-03-04"
  • search — Strips punctuation, trims query strings
  • text / notes / textarea — Preserves sentences, punctuation, paragraph structure

Error Codes

Handle errors via the onError callback on ConsciousProvider:

const handleError = (error: ConsciousError) => {
  switch (error.code) {
    case 'MIC_DENIED':          // User blocked microphone permission
    case 'STT_ERROR':           // Backend proxy or Wispr Flow API failure
    case 'AUTH_FAILED':         // Proxy rejected the request (check your API key)
    case 'CONNECTION_TIMEOUT':  // Proxy did not respond within connectionTimeoutMs
    case 'QUOTA_EXCEEDED':      // Monthly session limit reached on current plan
    case 'UNSUPPORTED_BROWSER': // WebRTC not available (IE, older Safari)
  }
}

On MIC_DENIED, the field silently falls back to normal text input. The user sees no error — the Conscious ring around the field simply disappears.

Trigger Keys

The default trigger is Fn — the interaction model popularised by Wispr Flow, which is training this muscle memory into millions of users. You can override it:

<ConsciousProvider triggerKey="fn">   // Fn key (default) — matches Wispr Flow
<ConsciousProvider triggerKey="ctrl"> // Left Ctrl — more discoverable cross-platform
<ConsciousProvider triggerKey="alt">  // Alt / Option — avoids Fn key macOS quirks

macOS note: Fn requires users to set System Preferences → Keyboard → Press Fn key to: Do Nothing, otherwise Fn toggles function keys. For consumer-facing apps, ctrl is typically safer.

CRM Integration Guide

CRM apps see the highest ROI from Conscious because sales reps are the canonical "don't want to type after a 45-minute call" user. The fields with the most impact:

  • Call notes and activity logs
  • Deal descriptions and next steps
  • Account notes and contact context
  • Support ticket summaries
  • Forecast notes / deal qualifications

Always set fieldLabel on these inputs — it's passed to the STT engine and improves CCS by ~12% for unlabelled fields.

Security Architecture

Audio travels from the browser to your backend proxy via an encrypted WebSocket. Your proxy relays it to the STT provider server-to-server. The API key lives only on your server — it never appears in network inspector, the client bundle, or any browser context.

Conscious stores zero bytes of audio after transcription. Audio is processed and discarded from our infrastructure within milliseconds of the session ending. Nothing persists.

White-label proxy URLs mean users see api.yourapp.com — never a Conscious or Wispr domain. Your brand, end to end. SOC 2 Type II is on the Q4 roadmap; BAA available for healthcare on Enterprise.

FAQ

Does this work with controlled React components?
Yes. Conscious fires native input and change events on commit, so React controlled components (value + onChange) work with zero modification.

What happens if the microphone is denied?
Conscious calls your onError handler with MIC_DENIED, then silently falls back. The field works exactly as before — the user just doesn't get voice input.

Will my users know they're using Conscious?
Only the subtle green ring on the active field. There's no Conscious branding in the UI — the whole experience is white-labelled under your product. Users see "voice input powered by YourApp" not "powered by Conscious."

Is the 95% accuracy guarantee per session or rolling?
It's measured as Word Error Rate over a rolling 30-day window across all sessions in your workspace. If aggregate WER exceeds 5%, that month is refunded. No questions asked — it's in the contract.

Can I use Conscious with non-React apps?
Not yet. React is the only supported framework in Phase 1. A Vue adapter and vanilla JS wrapper are on the Phase 2 roadmap (Q3 2025).
